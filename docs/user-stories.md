# User Stories - Theme Workflow

## Story 1: Designer with Brand Guidelines
- **As a** BI developer with strict brand guidelines
- **I want to** extract design system from our brand materials
- **So that** my dashboards match our company's visual identity

## Story 2: Inspiration-Driven Designer  
- **As a** dashboard creator who found a beautiful design online
- **I want to** capture that aesthetic and apply it to my Power BI work
- **So that** I can create professional-looking dashboards without starting from scratch

## Story 3: Multi-Project Consistency
- **As a** consultant working on multiple client projects
- **I want to** save each client's design theme once
- **So that** all dashboards for that client have consistent branding

## User Requirements

1. **Theme Editability:** Users can edit saved themes
2. **Theme Versioning:** Users can save multiple versions (duplicate with timestamp)
3. **Theme Sharing:** Users can:
   - Share project/link from Ditto
   - Export theme as JSON file
   - Import JSON file to add to library
4. **Theme Overrides:** When applying theme to prototype, users can override individual values
5. **AI Suggestions:** Not needed for MVP (future consideration)
